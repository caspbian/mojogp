# MojoGP 0.26.6.0 Release Notes

MojoGP `0.26.6.0` is the first PyPI-oriented packaging release for the public source distribution.

## Highlights

- Adds the MIT license and package metadata for the base `mojogp` distribution.
- Adds version-locked CUDA accelerator packages for `sm75`, `sm80`, `sm86`, `sm89`, `sm90`, `sm100`, and `sm120`.
- Loads native engines from matching `mojogp-cuda-smXX` packages with clear missing-package and version-mismatch errors.
- Pins Python support to 3.10 and 3.11, with Mojo `0.25.7.0` and MAX `25.7.0`.
- Keeps the README as a concise install and quickstart entry point, with detailed accelerator docs under Getting Started.

## Install

```bash
pip install "mojogp[sm89]"
```

Use the accelerator extra that matches the target GPU. The base `mojogp`
package does not auto-install a CUDA accelerator package. See
[Accelerator Targets](../getting-started/accelerator-targets.md) for the target
table. Underscored aliases such as `pip install "mojogp[sm_89]"` also resolve to
the matching accelerator package.

## Validation

Release preparation evidence currently includes strict docs build plus local `sm89` build, unit, integration, and minimal system validation.

The other published accelerator packages are available for matching GPUs, but require package install smoke and hardware fit/predict smoke evidence before they should be described as tested on that hardware.
